# CONTENT-RULES Rule 5: document frame-story slot reservation

## Context

T-1777564800228 added a frame-story slot carve-out in
`selectTrackStorylets`: when any track marked `category = 'frame_story'`
has an eligible candidate, one of the `maxStorylets` global slots is
reserved for it; the remaining slot(s) are filled from `life_stream`
tracks by urgency. The `opportunity` track is currently the only frame-
story track in the chapter.

CONTENT-RULES Rule 5 is the right place to document this for content
authors — it affects how they reason about competition between tracks. A
storylet on the opportunity track is no longer outranked by short-window
life-stream urgencies; conversely, life-stream tracks have effectively
one fewer slot when frame-story has eligible content.

## Acceptance criteria

- [ ] Update CONTENT-RULES.md Rule 5 (or whichever rule covers track
      scheduling / pool selection) to describe:
  - Frame-story tracks get a reserved slot when they have an eligible candidate
  - Life-stream tracks compete for the remaining `maxStorylets - 1` slot(s)
  - The reservation is not held open if no frame-story candidate exists
  - Currently, only `opportunity` is `category=frame_story`
- [ ] Add a worked example showing how a typical chapter-one PM resolves
      with the reservation in effect
- [ ] Cross-link to `src/core/tracks/selectTrackStorylets.ts` for the
      authoritative implementation

## Out of scope

- Changing the scheduling policy
- Re-categorizing other tracks as frame-story

## Files of interest
