# Content Studio JSONB-replace vs merge — choices field clobbers migration patches

## Context

While diagnosing T-1777564800228 (terminal scheduler bug), found that
`glenn_pastime_paradise.choices[0].sets_flag` was missing in the live DB
despite migration `20260417100001_add_sets_flag_to_choices.sql` explicitly
patching it. The row's `updated_at` was `2026-04-23T12:07:26Z` — six days
after the patch. Something on or near 04-23 overwrote the `choices` array
and dropped the migration-added field.

The clobber was specific:
- `glenn_pastime_paradise.choices[0]` (head_to_evening) → `sets_flag` lost
- `terminal_first_visit.choices[0]` (leave_terminal) → `sets_flag` intact
  (also patched by the same 04-17 migration)

This pattern strongly suggests Content Studio editing the storylet rewrote
the choices array from its in-memory model and dropped fields it didn't
know about. The 04-17 migration added `sets_flag` to existing choices via
`jsonb_set`, but the editor's "save" path may do a full replace rather than
a deep merge — silently discarding migration-applied keys.

## Why this is critical

This bug just patched a single symptom. The mechanism may still be active.
Any choice-level field added by a migration AFTER a storylet's initial
creation is at risk:
- `sets_flag`, `precludes`, `set_npc_memory`, `practices_skills`,
  `period_stance`, `events_emitted`, `skill_modifier`, etc.
- Any future field added via `jsonb_set` migrations to existing rows.

If the Content Studio is the clobber source, a content author saving any
edit to glenn_pastime_paradise (or any other affected storylet) could
re-clobber the very fix we just shipped.

## Acceptance criteria

- [ ] Reproduce the clobber: edit a known-patched storylet via Content Studio

- [ ] Identify the save path: is it a JSONB replace or a deep merge?

- [ ] If JSONB replace: change to deep merge OR ensure the editor's
      in-memory model loads ALL fields from DB (including migration-added)
      before serializing back
- [ ] Add a regression test that saves a storylet via the editor's code path
      and asserts no fields are dropped
- [ ] Audit other choice-level fields for similar drops (links to T-1777574470937)
