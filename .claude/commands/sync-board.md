# Sync Board

Synchronize dashboard.html from TASKS.md (the single source of truth).

## When to Use

Run this after:
- Adding, removing, or moving tasks in TASKS.md
- Completing a design decision that spawns new dev tasks
- At the end of any session that modified TASKS.md
- At the start of a session to verify alignment

## Process

### Step 1: Run the sync script

```bash
node scripts/sync-dashboard.js
```

This parses TASKS.md and regenerates the `DEFAULT_TASKS` array inside dashboard.html.

### Step 2: Verify the output

Check the script output for task counts. Confirm they match what's in TASKS.md:
- Backlog count
- In Progress count
- Done count

### Step 3: Report to user

Tell the user:
- How many tasks in each column
- Remind them to click **"Reset to default"** in the browser if they have the dashboard open (localStorage overrides the defaults until reset)

## Design Decision Flow

When a design decision is resolved during a session:

1. **Update TASKS.md** — mark the design task as done, add any new concrete dev tasks to the appropriate backlog section
2. **Run `/sync-board`** — regenerates the dashboard
3. The new dev tasks are now visible in both TASKS.md and the dashboard

## Important Notes

- **TASKS.md is always the source of truth.** Never edit dashboard.html task data directly.
- The dashboard uses `localStorage` to track card positions. After syncing, the user must click "Reset to default" in the browser to pick up changes.
- Task IDs are auto-generated by the sync script — they don't need to be manually assigned.
- Section names in the dashboard come from `### Section Name` headers in TASKS.md.
